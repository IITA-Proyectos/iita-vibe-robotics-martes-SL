# FLL launch planning

## Recommended decomposition

Think in launch-return cycles rather than one giant run.

For each run record:

- launch position in base
- attachment installed
- first alignment reference
- missions attempted
- return trigger
- score if all works
- score if the last action fails

## Selection rules

- prioritize runs that score even if the final sub-step fails
- combine missions only if they share a corridor or attachment naturally
- penalize runs that require too much table-side setup precision

## Practice structure

1. single mission repetitions
2. full run repetitions
3. match-order repetitions with real attachment changes
