# WRO run planning

## Mission ranking matrix

| mission | points | travel | attachment burden | risk | notes |
|---|---:|---:|---:|---:|---|

## Strategy principles

- take early points that also improve alignment confidence
- split fragile multi-step actions away from the first reliable run
- design around official randomizations, not best-case positions
- keep one version of the run that still scores well with a reduced attachment set

## Handover to coding

For each selected mission define:

- entry pose
- main action
- exit pose
- fallback if action fails
- whether the robot returns to a known reference edge or base
